## 推荐阅读路线

<CardGrid stagger>
	<Card title="01 入门：先把环境搭好" icon="setting">
		安装 Git、完成首次配置、理解 Git 的诞生背景与核心理念。
	</Card>
	<Card title="02 基础：掌握日常必备" icon="open-book">
		围绕 `add` / `commit` / `status` / `diff` 等高频命令建立正确心智模型。
	</Card>
	<Card title="03 分支：真正学会协作" icon="document">
		理解分支是怎么来的、如何合并、如何解决冲突，并形成稳定的协作习惯。
	</Card>
	<Card title="04 协作：远程与团队流程" icon="add-document">
		掌握远程仓库交互、拉取与推送、以及 GitHub PR 的标准流程。
	</Card>
	<Card title="05 高级：重写历史与排错" icon="pencil">
		深入 `rebase`、`reset`、`cherry-pick` 等高级操作，学会在复杂场景里做“正确的修复”。
	</Card>
	<Card title="06 原理：搞懂 .git 里发生了什么" icon="open-book">
		从对象模型与引用系统出发，理解 Git 为什么能这么快、这么可靠。
	</Card>

## 使用建议

- 建议按模块顺序阅读：01 → 02 → 03 是最短学习路径。
- 每一章都尽量配合命令行实操：复制示例命令、观察输出、再回头理解原理。
- 后续章节（07–09）更适合在有真实项目经验后作为“工具箱”随用随查。
